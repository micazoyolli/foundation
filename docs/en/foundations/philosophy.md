# Philosophy

Foundation shares what has already proven to be common across independent projects. Its value is being small, predictable and easy to remove if a project no longer needs it.

<div class="principle-grid">
  <div class="principle-card"><strong>Small</strong><span>The API grows only when a need appears in several real repositories.</span></div>
  <div class="principle-card"><strong>Composable</strong><span>Utilities can be used independently without adopting a full architecture.</span></div>
  <div class="principle-card"><strong>Framework-agnostic</strong><span>No React dependency and no visual runtime.</span></div>
  <div class="principle-card"><strong>Accessible</strong><span>Keyboard, focus and reduced-motion patterns are documented when repeated.</span></div>
  <div class="principle-card"><strong>Predictable</strong><span>Small functions, direct names and documented behavior.</span></div>
  <div class="principle-card"><strong>No brand assumptions</strong><span>No final colors, logos, copy, routes or project metadata.</span></div>
  <div class="principle-card"><strong>Built from production</strong><span>Extracted from real needs, not imagined components.</span></div>
</div>

## Belongs in Foundation

Base tokens, small SCSS mixins, framework-free TypeScript helpers, DOM/a11y primitives and SEO/build helpers without brand data.

## Does not belong in Foundation

Visual React components, buttons, cards, final layouts, brand typography, project metadata, business logic and one-off helpers.
